## Notes

- Permission names use the frozen `resource.action` convention; the decorator's type (`` `${string}.${string}` ``) rejects non-conforming names at compile time.
- The guard contains the documented **seam** for Priority 2 (ACTION-PLAN 2.2): once auth exists, it resolves the actor from the WS-14 request context and delegates to `policy.can(actor, permission)`. This task deliberately ships only the deny-by-default mechanism — no catalog, no roles, no sessions.
- Every future endpoint must carry `@RequirePermission` or (rarely, reviewed) `@Public()` — forgetting fails closed with a warning log naming the handler.
