## Implementation notes

- Rules live in `packages/config/eslint-plugin-hr.mjs`, exported as `@hr/config/eslint-plugin`.
- **`hr/module-boundaries`** resolves relative import specifiers to absolute paths and compares source/target module directories — this correctly distinguishes a module's own subfolders (`../application/...`) from sibling modules, which string-pattern approaches (`no-restricted-imports`) cannot. Only `…/modules/<target>/public-api` is allowed across modules.
- **`hr/rtl-safe-classes`** tokenizes `className`/`class` string literals and template-literal chunks, strips variant prefixes (`md:hover:` etc.), and errors on the physical-direction utility list (`pl-/pr-/ml-/mr-/left-/right-/inset-x-/text-left/text-right/float-left/float-right/border-l/-r/rounded-l/-r/-tl/-tr/-bl/-br/scroll-*`). Dynamic class expressions aren't statically checkable — those remain a review concern (visible escape hatch: an inline disable comment).
- Both deliberate violations were reverted after capture; the diff in the commit contains only the rule implementation and wiring.
