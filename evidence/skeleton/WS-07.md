### Notes

- **Lint conflict found and resolved:** `@typescript-eslint/consistent-type-imports` rewrites DI class imports to `import type`, which erases the decorator metadata NestJS DI depends on — silently breaking injection. The rule is disabled **for the API app only** (documented in `apps/api/eslint.config.mjs`); it stays active for web and packages.
- `example-consumer` is deliberately kept (not throwaway): it is the test subject for the WS-08 boundary lint rule (a deep import into `example/` internals must fail lint).
- `domain/` and `infra/` folders are documented but not created until needed — no empty ceremony.
- ADR-003 remains **Proposed** until WS-08 proves the boundary is mechanically enforced; then it flips to Accepted.
