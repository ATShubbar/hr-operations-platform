### CI failure found and fixed en route

First CI run was **red**: `SASL: SCRAM-SERVER-FIRST-MESSAGE: client password must be a string`. Root cause: **Turbo v2 strict environment mode strips undeclared env vars from task processes** — workflow-level `DATABASE_URL`/`STAFF_DATABASE_URL`/`CLIENT_DATABASE_URL` never reached vitest (locally masked by `dotenv` loading `.env` inside the process). Fix: declare them in `turbo.json` `globalEnv` (which also correctly keys turbo's cache on them). Verified by simulating CI locally (`.env` removed, env vars only): 24/24 pass. Second CI run: **green, owner-confirmed**.
