## Notes

- `actorId`/`clientId` are context placeholders populated by auth (Priority 2); every log line already carries the fields so the shape is stable from day one.
- Logger is deliberately dependency-free; swapping in pino later is contained to `json-logger.ts`.
- Flagged for the BullMQ task (post-skeleton): AsyncLocalStorage context does not cross queue boundaries — job processors must restore context from job payload (carry `requestId` in job data).
- Incident during evidence capture: killing a Node process whose stdout is file-redirected can drop buffered log lines — the trace was captured from the still-running instance instead. Harmless locally; irrelevant in production where logs stream to the collector.
